# 婴儿图式 / Kindchenschema

## 目的

根据可见的婴儿化形态或幼态延续形态估计情感。婴儿图式特征通常会引发可爱感、温柔感、接近倾向以及照护动机。当形态而非故事情节驱动情感时，该技能适用于人类婴儿、幼年动物，以及经过刻意幼态化设计的角色或产品。

## 路由卡

适用情形：
- 画面中的主要主体是人类婴儿、幼年动物，或经过刻意幼态化设计的角色／物体。
- 情感表现为可爱、温柔、保护欲、温暖感或照护动机。
- 婴儿化形态比周围情境或表情更重要。

不适用情形：
- 主体是表情中性或轻微的成年人面孔；应使用 `todorov-face-evaluation`。
- 婴儿化主体明显处于危险、痛苦、疼痛、疾病或被利用的状态；应使用 `cognitive-appraisal`。
- 情感来源于诡异的不协调感、身体姿势、污染、捕食性威胁或审美风格，而不是婴儿图式。

视觉触发因素：
- 较大的头身比例、较高的前额、圆润的脸部、较大且位置偏低的眼睛。
- 丰满的面颊、较小的鼻子／嘴巴、较短的四肢、圆润的身体、柔软且类似毛绒玩具的身体比例。
- 婴儿、幼兽、类似幼犬／幼猫的形态，以及卡哇伊／Q版／幼态延续设计。

近似但不匹配的边界：
- 具有娃娃脸的成年人肖像 -> 除非明确采用了婴儿化设计，否则应使用 `todorov-face-evaluation`。
- 处于危险或痛苦中的可爱主体 -> `cognitive-appraisal`。
- 具有可爱外观的动物，但捕食者／威胁形态占主导地位 -> `evolved-fear-module`。

## 核心构念

- **婴儿图式负载**：婴儿化特征的数量和强度。
- **可爱感知**：观看者可能产生“可爱”“讨人喜欢”或温柔的反应。
- **照护动机**：接近、保护、养育、抱持或帮助。
- **情境安全性**：当主体看起来安全且无害时，可爱反应最强。
- **诡异感或痛苦覆盖效应**：令人毛骨悚然、受伤、被利用或处于危险中的主体，可能使效价发生反转。

## 适用条件

当形态是情感来源时，使用该技能。关键问题是：产生照护／可爱反应的是否是可见的身体比例，而不是叙事情境。

## 不适用规则

- 不要将该技能用于成年人面孔评价，但可以对较弱或部分幼态延续特征进行说明。
- 当危险、痛苦或被利用状态占主导地位时，不要将脆弱主体评价为“可爱”。
- 不要将该技能用于动物威胁、污染或由姿势驱动的情绪。
- 不要根据外表推断真实的健康状况、发育状态、人格或价值。

## 适用程度

强：
- 多种婴儿图式特征清晰可见，并且情境安全／无害。

部分适用：
- 存在一些幼态延续线索，但年龄、物种、设计或情境削弱了这种效果。

弱：
- 可爱感主要是通过文化习得、风格化设计产生的，或者相对于其他情感机制处于次要地位。

## 推理步骤

1. 识别主体类型：婴儿、幼年动物、幼态延续设计，或具有较弱幼态延续特征的成年人。
2. 列出可见的婴儿图式特征，并估计其负载。
3. 检查是否存在安全性、痛苦、危险、被利用状态或诡异感覆盖效应。
4. 估计可能产生的可爱／照护反应。
5. 将安全情境中的婴儿图式映射为正效价和低至中等唤醒度。
6. 当形态不是主要的情感驱动因素时，将适用程度标记为部分或较弱。

## VA 映射

| 情形 | 效价 | 唤醒度 |
|---|---:|---:|
| 高婴儿图式负载，安全／无害 | 8.0-9.2 | 3.5-5.8 |
| 中等婴儿图式负载，安全／无害 | 7.0-8.2 | 3.0-5.0 |
| 轻微／部分婴儿图式 | 6.0-7.2 | 3.0-5.0 |
| 极度可爱／可爱侵略 | 8.5-9.5 | 5.0-6.8 |
| 痛苦、危险或诡异感覆盖效应 | 2.0-5.0 | 4.5-7.5 |
| 与婴儿图式的匹配较弱 | 保持接近直接 VA 结果 | 保持保守估计 |

## 边界说明

- 婴儿图式关注的是形态，而不仅仅是年龄较小或内容积极。
- 安全性会强烈影响可爱反应。
- 如果经过设计的角色具有可见的幼态延续比例，也可以适用该技能。
- 如果没有任何专门化线索能够明显匹配，则应路由到 `no_specialized_skill`。

## 输出格式

实验提示词会将该分析转换为 JSON。应包含顶层字段 `valence_score`、`arousal_score` 和 `applicability`。在分析理由中，应报告主体类型、婴儿图式特征、负载、情境覆盖效应检查结果、照护／可爱反应以及 VA 映射。
