# 婴儿图式／Kindchenschema

## 适用性门槛

必须满足：
- 占主导地位的主体是人类婴儿、幼年动物，或经过刻意幼态化设计的角色／物体。
- 可以看到多个婴儿化比例特征，并且这些特征能够合理地驱动温柔感或照料倾向。
- 在考虑周围情境之后，形态特征在情感上仍然具有重要作用。

排除：
- 成人面孔、普通的小型物体，或仅在文化上被标记为“可爱”但缺乏幼态结构的主体。
- 捕食性形态、污染、身体动作或社会事件是主要机制。
- 严重的痛苦、危险、疾病或诡异的不协调感完全决定了情感；此时只将婴儿图式保留为修正因素。

近似但不适用：
- 具有表情的成人肖像 -> facial-expression-affect；中性成人肖像 -> none。
- 具有可信攻击威胁的幼年动物 -> evolved-fear-module。
- 明显处于危险中或正在受苦的婴儿 -> cognitive-appraisal 成为主要机制。

## 视觉变量

- **头部／身体比例**：相对较大且圆润的头部，以及短小紧凑的身体。
- **面部构型**：高额头、圆脸、较大且位置偏低的眼睛、丰满的脸颊，以及较小的鼻子／嘴巴。
- **四肢／身体形态**：较短的四肢、圆润的躯干、柔和的轮廓，以及较低的棱角感。
- **真实感／设计形式**：真实的婴儿／幼年动物，或毛绒玩具、Q 版角色、玩偶或机器人。
- **安全性及覆盖因素**：放松姿态、照料接触、玩耍、睡眠，与哭泣、受伤、束缚、威胁或诡异的类人外观之间的对比。

## 推断流程

1. 确认主体符合条件，并且只列出能够直接看到的婴儿化比例特征。
2. 将各项特征整合为低／中／高图式负载；仅仅属于某一类别并不足以完成判断。
3. 确认该形态支持接近／照料，而不是威胁或回避。
4. 检查安全性、痛苦、利用性情境，以及诡异感带来的覆盖效应。
5. 使用锚定范围判断 VA；婴儿图式提供积极先验，具体情境决定所采用的分支。

## 唤醒度门槛

可爱感是温暖且低激活的：处于休息状态或轻柔摆姿的主体通常位于 3.4-4.6。
只有在能够看到充满活力的玩耍、主动接近或痛苦时，才将唤醒度提高到 5 以上。
大眼睛和柔软的毛发并不是激活线索。不存在激活线索 -> 唤醒度 <= 4.5。

## VA 锚定范围（1-9）

| 情况 | 效价 | 唤醒度 |
|---|---:|---:|
| 高图式负载，安全／良性情境 | 7.2-8.2 | 3.8-5.0 |
| 中等图式负载，安全情境 | 6.5-7.4 | 3.4-4.6 |
| 具有清晰婴儿图式的活跃玩耍 | 7.0-8.0 | 5.0-5.8 |
| 痛苦／危险／诡异感产生覆盖效应 | 3.0-4.5 | 5.0-6.0 |
| 匹配度较弱 | 5.2-6.2 | 3.5-4.5 |

这些锚定范围是典型区间，而不是强制输出值；应在区间内进行插值。当证据不足时，
应停留在该区间中最接近中性值的一端。

## 示例

- **平静型**：一只具有大而圆的头部、大眼睛和短腿的小狗，在柔和的光线下睡在毯子上。
  -> 多项比例特征相互汇聚、睡眠、良性情境；activation_evidence []。
  **V 7.6，A 3.6**，证据强。
- **活跃型**：一名正在大笑的幼儿快速爬向镜头，同时伸出手臂。-> 高图式
  负载，加上可见的活跃接近行为；activation_evidence ["快速爬行", "伸手
  动作"]。**V 7.8，A 5.4**，证据强。

## 输出约定

在共享 JSON 中分别报告可见比例特征、图式负载、照料倾向、
安全性／覆盖因素线索，以及顶层 `activation_evidence`（如果不存在则为空列表）。
绝不要推断健康状况、价值高低或实际脆弱性。
