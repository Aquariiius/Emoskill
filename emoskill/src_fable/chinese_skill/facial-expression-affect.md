# 面部表情情感

## 适用性门槛

必须满足：
- 人类面孔是占主导地位的主体，并且足够大、足够清晰，可以进行辨读。
- 眉毛／眼睛区域与嘴巴／下颌区域分别独立支持同一种瞬时表情。
- 承载情感的是面部表情，而不是某个事件、物体或身体动作。

排除：
- 中性、静止或含义模糊的面孔；结构性的第一印象不属于本技能的范围。
- 手或物体按压、遮挡或明显使某个诊断区域发生变形（external_distortion）。
- 会产生后果的事件或互动决定情感 -> cognitive-appraisal；全身信号 -> emotional-body-language；婴儿化形态 -> baby-schema。

近似但不适用：
- 处于更大且有意义事件中的微笑人物 -> 如果事件占主导，则使用 cognitive-appraisal。
- 哭泣／大笑的人群，但没有任何一张单独面孔能够清晰辨读 -> emotional-body-language。

## 视觉变量

- **面孔可靠性**：face_clear、brow_eye_visible、mouth_jaw_visible、external_distortion——全部记录为可观察的布尔值。
- **区域证据**：眉毛位置、眼睛张开程度、嘴部曲率／张开程度、下颌紧张程度；分别记录。
- **一致性／强度**：相互独立的区域是否一致，以及表情表现有多强（轻微、清晰、极端）？
- **观察者传递**：可见的身体／情境是否支持观察者共享这种情感（增强因素／削弱因素）？

## 推断流程

1. 首先填写 face_reliability 布尔值；任何一个诊断区域为 false，或存在变形 -> gate_decision 使用 use_direct。
2. 在不使用情绪标签的情况下，记录眉毛／眼睛区域以及嘴巴／下颌区域的观察结果。
3. 推断主要表情以及一个备选表情；要求区域之间保持一致。
4. 估计表情强度和 viewer_transfer（低／中／高等级，并记录增强因素／削弱因素）。
5. 如果门槛通过，则使用锚定范围判断 VA；否则输出保守分数并使用 use_direct。

## 唤醒度门槛

表情的效价比表情强度更容易传递：摆拍式微笑属于平静内容。
只有在高强度表情同时具有可见激活线索时，才将唤醒度提高到 5 以上（张口
大笑、尖叫、睁大双眼的恐惧、愤怒时的紧张）。轻微微笑、安静的悲伤和摆拍式
肖像都不是激活线索。不存在激活线索 -> 唤醒度 <= 4.5。

## VA 锚定范围（1-9）

| 表情 | 效价 | 唤醒度 |
|---|---:|---:|
| 清晰微笑、放松的快乐 | 6.8-7.8 | 4.4-5.4 |
| 张口大笑、兴奋 | 7.2-8.2 | 5.4-6.4 |
| 悲伤、安静哭泣 | 2.6-3.6 | 4.2-5.2 |
| 愤怒、可见的暴怒紧张 | 2.8-3.8 | 5.6-6.6 |
| 恐惧、睁大双眼的警觉 | 3.0-4.0 | 6.0-7.0 |
| 轻微／含义模糊的表情（如果不确定，则转入 use_direct） | 4.5-6.0 | 4.0-5.0 |

这些锚定范围是典型区间，而不是强制输出值；应在区间内进行插值。当证据不足时，应停留在该
区间中最接近中性值的一端。

## 示例

- **高激活型**：一名女性张口大笑，面颊上扬，眼角出现笑纹，并且面孔占满
  画面。-> 高强度且一致的快乐，两个区域相互一致；activation_evidence ["张口
  大笑"]。gate_decision use_skill。**V 7.6，A 6.0**，证据强。
- **平静型**：一名男性低头，眉头收紧，嘴角下垂，脸颊上有一滴泪。->
  一致且安静的悲伤，没有动作或紧迫性；activation_evidence []。gate_decision
  use_skill。**V 3.0，A 4.6**，证据强。

## 输出约定

共享 JSON 必须包含 face_reliability（四个布尔值）、viewer_transfer
（level／amplifiers／attenuators）、gate_decision（use_skill|use_direct），以及顶层
`activation_evidence`（如果不存在则为空列表）。报告感知到的表情，绝不要报告身份、
人格或该人物真实的内在状态。
