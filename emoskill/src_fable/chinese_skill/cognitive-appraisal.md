# 认知评估

## 适用性门槛

必须满足：
- 存在可见且会产生后果的变化或利害关系：伤害、危险、损失、阻碍、救援、成就、关爱、亲密或违反规范。
- 如果不对情境进行解释，就无法通过单一的直接机制读取情感。
- 至少有两个评估维度具有相互独立的可见证据。

排除（转至 no_specialized_skill，而不是此技能）：
- 人物只是出现在画面中、摆姿势、站立、进食、购物、工作或进行没有可见后果的日常活动。
- 某种直接机制占据主导地位：面部表情、身体动作、污染、威胁性动物、婴儿图式、恢复性、广阔尺度或构图。
- 相关解释需要虚构过去经历、意图、同意情况或人格特征。

近似但不适用：
- 清晰的姿态信号，并且对情境依赖很小 -> emotional-body-language。
- 可辨读的表现性面孔 -> facial-expression-affect。
- 当行动主体和后果可见时，武器／事故／暴力仍归入此技能。

## 视觉变量

- **观察记录表**：可见的人物、物体、动作、接触、场景和后果；不记录心理状态。
- **目标一致性**：事件是否明显帮助某人（关爱、成功、照料）或伤害某人（受伤、损失、攻击）。
- **行动主体／控制力**：谁在行动、谁受到影响，以及是否存在逃离或应对选项。
- **确定性／迫近性**：事件正在发生还是已经完成；距离影响发生还剩多少时间的线索。
- **事件极性检查（必须执行）**：在评分之前判断事件是有利还是有害；拥抱、亲吻、庆祝和救援属于积极事件。

## 推断流程

1. 只根据可见事实建立观察记录表。
2. 说明会产生后果的变化。如果无法明确指出这种变化，则输出适用性较弱并推迟判断。
3. 使用所引用的证据对事件极性进行分类（积极／消极／模糊）。
4. 只估计得到证据支持的维度；当确实存在模糊性时，给出两个假设。
5. 使用相应分支的锚定范围判断 VA；正在发生的危险具有激活性，而事后场景不具有激活性。

## 唤醒度门槛

唤醒度取决于迫近性和紧迫性，而不是事件意义的严重程度：正在发生的攻击具有较高唤醒度；
已经结束的事故、悲伤场景、葬礼或事后照片具有低至中等唤醒度（3.5-5.2）。悲伤或
严肃的意义并不是激活线索。不存在激活线索 -> 唤醒度 <= 4.5。

## VA 锚定范围（1-9）

| 分支 | 效价 | 唤醒度 |
|---|---:|---:|
| 正在发生的持械攻击／威胁或暴力事件 | 2.0-3.0 | 6.0-7.5 |
| 事后场景：已经发生的事故、损失、悲伤 | 2.5-3.5 | 4.0-5.2 |
| 亲和／照料：拥抱、安慰、家庭温暖 | 6.5-7.5 | 4.2-5.4 |
| 亲密／情色伴侣，具有双方同意的呈现方式 | 6.5-7.5 | 5.4-6.4 |
| 成就／庆祝结果 | 6.5-7.5 | 5.0-6.0 |
| 利害关系模糊（此时更倾向于推迟判断） | 4.5-5.5 | 4.0-5.0 |

这些锚定范围是典型区间，而不是强制输出值；应在区间内进行插值。当证据不足时，应停留在该
区间中最接近中性值的一端。

## 示例

- **消极事件**：一人在室内用手枪瞄准一名蜷缩畏缩的受害者。-> 伤害正在发生、行动主体明确、
  没有可见的逃离途径；activation_evidence ["瞄准的武器", "蜷缩畏缩的受害者"]。
  **V 2.4，A 6.9**，证据强。
- **积极事件**：两个人在机场到达口紧紧拥抱，旁边放着行李。->
  重聚事件、获得亲和关系、双方相互接近；activation_evidence ["主动拥抱"]。
  **V 7.2，A 5.0**，证据强。

## 输出约定

包括观察记录表、事件极性、得到支持的维度、竞争性假设，以及
顶层 `activation_evidence`（如果不存在则为空列表）。绝不要将虚构的意图作为事实报告。
